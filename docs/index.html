Projects

The Hub

The Hub provides HCA Healthcare with a democratized enterprise architecture platform, collaboratively contributed to by the teams and stakeholders it serves. By creating a relational map of products, their associated technical details, and key members, a higher level of transparency and knowledge sharing has increased operational efficiency. Centrally locating key technical product information renders the governance process more effective and efficient.

The platform integrates the enterprise API gateway and inner-sourced code libraries. There are more DevOps and DevSecOps efforts to integrate into the platform on the horizon.

The architecture is composed as a full PaaS implementation in Azure. CosmosDB handles data storage and change feed functionality. Azure Functions make up the API layer in a microservice architecture. Azure Frontdoor acts the WAF. App Insights is configured for alerting and monitoring across the stack.

HCA Healthcare Design System

HCA Healthcare's Design System Neutron is built to scale to meet the needs of teams and brands across the HCA Healthcare enterprise. The intention for this tool is to ensure consistency of design and UI across all products for the enterprise, enable rapid development for dev teams, and meets Accessibility needs of end users.

As a member of Enterprise Architecture, I worked with the internal Cloud team and the UX Center of Excellence to create model for hosting and distributing the system, as well as an inner-sourcing model in collaboration with HCA's Evolve Development Committee where developers could contribute to the system.

Online Appointment Scheduler

The Online Appointment Scheduler is an application that allows patients to schedule an appointment with a practicing physician with the ease of sorting by name, date, time, specialty, and gender. This application was designed to remove the confusion and friction that users experience when trying to find a doctor whose practice correlates to their condition at a time that works for both parties.

Both the services and the frontend were developed and deployed in OpenShift containers. This project represented a step into modern development practices and tooling for this organization in our IT department. Working with system administrators, API developers, and frontend developers introduced a wealth of information and practice that was both challenging and rewarding.
